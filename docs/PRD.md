## 1. 產品概述

### 1.1 產品目標
透過遊戲化學習，讓 4-12 歲兒童快樂學習英文單字，同時收集潛在客戶（家長）資料供 OiKID 業務跟進。

### 1.2 目標用戶
- **主要用戶：** 4-12 歲兒童
- **次要用戶：** 家長（提供聯絡資訊）

### 1.3 核心價值
- 🎮 遊戲化學習，提高學習動機
- 📊 程度分析報告，了解孩子英文程度
- 📱 100% 觸控操作，兒童友善

### 2.2 遊戲規格

| 項目 | 規格 |
|------|------|
| 每場題數 | 30 題 |
| 題型 | 單字題 + 句子題（混合） |
| 操作方式 | **100% 觸控點選，無鍵盤輸入** |
| 答題選項 | 4 選 1 |
| 難度分佈 | 簡單 / 中等 / 困難 混合出題 |

### 2.3 題型設計

#### 題型 A：單字題
- **英翻中：** 顯示英文單字，選擇正確中文
- **中翻英：** 顯示中文，選擇正確英文單字

#### 題型 B：句子題
- **句子翻譯：** 顯示英文短句，選擇正確中文翻譯
- **句子：** 最多 10 個單字，生活情境用語

#### 難度定義
| 難度 | 單字特徵 | 比例 |
|------|----------|------|
| 簡單 | 3-5 字母 | 40% |
| 中等 | 6-8 字母 | 35% |
| 困難 | 9+ 字母 / 片語 / 句子 | 25% |

### 2.4 單字資料庫

| 等級 | 數量 | 狀態 |
|------|------|------|
| 國小 300 字 | 204 字 ✅ | 已完成 |
| 國中進階 800 字 | 500 字 ✅ | 已完成 |
| 生活短句 | 100+ 句 ⏳ | 待製作 |
| **總計** | **704 字 + 句子** | |

### 2.5 程度分析報告

報告內容：
1. **總體表現**
   - 答對題數 / 總題數
   - 正確率 %
   - 完成時間

2. **難度分析**
   - 簡單題正確率
   - 中等題正確率
   - 困難題正確率

3. **分類分析**
   - 強項分類（正確率 > 80%）
   - 弱項分類（正確率 < 50%）

4. **程度判定**
   - 國小初級（1-2 年級程度）
   - 國小中級（3-4 年級程度）
   - 國小高級（5-6 年級程度）
   - 國中程度

5. **學習建議**
   - 根據弱項給予建議

### 3.2 資料模型

```typescript
// 小孩資料
interface ChildInfo {
  name: string;        // 小孩名字
  age: number;         // 年齡 4-12
}

// 家長資料
interface ParentInfo {
  name: string;        // 家長姓名
  phone: string;       // 手機號碼
}

// 遊戲紀錄
interface GameSession {
  childInfo: ChildInfo;
  parentInfo?: ParentInfo;
  startTime: number;
  endTime?: number;
  questions: QuestionResult[];
}

// 單題結果
interface QuestionResult {
  questionId: number;
  type: 'word' | 'sentence';
  difficulty: 'easy' | 'medium' | 'hard';
  category: string;
  isCorrect: boolean;
  timeSpent: number;   // 毫秒
}

// 程度報告
interface LevelReport {
  totalScore: number;
  accuracy: number;
  difficultyBreakdown: {
    easy: { correct: number; total: number };
    medium: { correct: number; total: number };
    hard: { correct: number; total: number };
  };
  categoryStrengths: string[];   // 強項分類
  categoryWeaknesses: string[];  // 弱項分類
  level: 'elementary-1' | 'elementary-2' | 'elementary-3' | 'junior-high';
  recommendations: string[];
}
```

## 4. UI/UX 規範

### 4.1 設計原則

1. **活潑溫暖：** 整體風格活潑但溫暖，不冷硬
2. **圓角設計：** 所有按鈕、卡片、選項都使用圓角（16-24px），禁止尖銳角度
3. **大按鈕：** 最小 48x48px，建議 64x64px 以上
4. **即時回饋：** 點擊後 100ms 內有視覺回饋
5. **獎勵動畫：** 答對有星星/煙火/彩帶效果
6. **無打字：** 遊戲過程 100% 點選操作
7. **兒童友善：** 不是工程師功能型遊戲，注重視覺感受

### 4.2 色彩規範（暖色系）

| 用途 | 顏色 | Hex |
|------|------|-----|
| 主色（橘） | 活潑橘 | #FF9F43 |
| 輔色（藍） | 天空藍 | #54A0FF |
| 輔色（粉） | 溫柔粉 | #FF6B9D |
| 輔色（綠） | 草地綠 | #5AD07A |
| 正確回饋 | 開心綠 | #26DE81 |
| 錯誤回饋 | 柔和紅 | #FC5C65 |
| 背景色 | 奶油白 | #FFF9F0 |
| 卡片背景 | 純白 | #FFFFFF |
| 主要文字 | 深咖啡 | #2D3436 |
| 次要文字 | 暖灰 | #636E72 |

### 4.3 視覺風格

```
┌─────────────────────────────────────┐
│  ✨ 整體風格關鍵字 ✨                │
│                                     │
│  • 圓潤可愛                         │
│  • 溫暖明亮                         │
│  • 活潑有趣                         │
│  • 鼓勵正向                         │
│  • 減少壓力感                       │
└─────────────────────────────────────┘
```

**按鈕樣式：**
- 圓角：16-24px
- 陰影：柔和投影（0 4px 12px rgba(0,0,0,0.1)）
- 點擊效果：輕微縮放 + 顏色變化
- 禁用狀態：降低透明度，不使用灰色

**選項卡片：**
- 圓角：20px
- 邊框：2px 柔和邊框
- 選中狀態：邊框加粗 + 背景色變化
- 懸停效果：輕微上浮

**動畫效果：**
- 答對：星星飛舞 + 「太棒了！」文字
- 答錯：輕微搖晃 + 「再試試！」鼓勵
- 轉場：柔和淡入淡出（300ms）

### 4.4 表單輸入

**名字輸入（小孩/家長）：**
- 支援中文、英文輸入
- 使用手機內建鍵盤
- 輸入框圓角設計
- 清除按鈕方便修改

### 4.3 響應式設計

| 裝置 | 寬度 | 佈局 |
|------|------|------|
| 手機 | < 640px | 單欄，全螢幕 |
| 平板 | 640-1024px | 置中卡片 |
| 桌面 | > 1024px | 置中卡片，最大 500px |
